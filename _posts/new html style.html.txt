paper is difficult. Forty-six preliminary definitions, together with several important preliminary propositions, must be mastered before the main results are reached.” – Nagel, E. and Douglas R. Hofstadter. 2001. “Gödel’s proof, Rev. ed.” New York: New York University Press.<br></em><br>But fear not, we can grasp the essential idea behind it by constructing a “Mini Gödelian Machine”.<br><br><u><strong>A Mini Gödelian Machine</strong></u>[1]<br></span></p><p><span style="font-family:times new roman,times,serif;">Let us consider a computing machine that prints out various expressions composed of the following five symbols: <em>~ P N ( )</em>​.<br><br>An expression <em>X</em> is called <em>printable</em> if the machine can print it. We assume the machine is so programmed that any expression the machine can print will be printed sooner or later.<br><br>By the <em>norm</em> of an expression <em>X</em>, we mean the expression <em>X(X)</em>, e.g. the norm of <em>X: John is Reading</em> is <em>N(X): John is Reading(John is Reading)</em>. John is reading, “John is reading” and is therefore not self-referential. By a <em>sentence</em> we mean any expression of one of the following four forms (where <em>X</em> is any expression whatever).<br><em>(1) P(X)<br>(2) ~P(X)</em><br><em>(3) PN(X)</em>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [Read: the norm of <em>X</em> is printable]<br><em>(4) ~PN(X)</em>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; [Read: the norm of <em>X</em> is not printable]<br><br>Informally, <em>P</em> stands for “printable”, <em>~</em> stands for not, and <em>N</em> stands for “the norm of”. And so we define <em>P(X)</em> to be true iff <em>X</em> is printable, <em>~P(X)</em> to be true iff <em>X</em> is not printable, <em>PN(X) </em>to be true iff the norm of <em>X</em> is printable, and <em>~PN(X)</em> to be true iff the norm of <em>X</em> is not printable.<br><br>We have now given a perfectly precise definition of what it means for a sentence to be true. We now assume that we are also given that the machine is completely accurate in that all sentences printed by the machine are true. It never prints any false sentences. Thus, for example, if the machine ever prints <em>P(X)</em>, then <em>X</em> is really printable, and so the machine will print <em>X</em> sooner or later.<br><br>What about the converse? If <em>X</em> is printable, does it follow that <em>P(X)</em> is also printable? Not necessarily. If <em>X</em> is printable, then <em>P(X)</em> is certainly <em>true</em>, but that does not mean that <em>P(X)</em> must be printable. We are given that all printable sentences are true, but we are not given that all true sentences are printable! As a matter of fact, there is a sentence&nbsp;which is true but not printable: <em>~PN(~PN)</em>&nbsp;[Read: It is not possible to print the norm of this expression(It is not possible to print the norm of this expression]! The norm of <em>~PN </em>is the very sentence <em>~PN(~PN)</em> and so the sentence is true iff it is not printable, or it's printable and not true. The latter alternative violates the given hypothesis that the machine never prints sentences that are not true. Hence the sentence must be true but the machine cannot print it.<br><br>Gödel took this insight and went about constructing the provability of propositions. What Gödel did was showed that within any axiomatic system, there would remain true propositions which cannot be proven within that system.<br><br><u><strong>How he did it</strong></u>[2]<br>As a first step, Gödel showed how to assign a unique number, called the <em>Gödel number</em>, to every string of symbols in a system. This numbering allowed him to uniquely number not only random strings but also statements, theorems, and even entire arguments, valid or otherwise. For example, all these strings could be represented:<br>4 + 9 = 13<br>4 + 9 = 1962<br>∀x ∃y[x + y = 3]<br>=48&lt;+33-=7=<br></span></p><p><span style="font-family:times new roman,times,serif;"><br></span></p><p><span style="font-family:times new roman,times,serif;">Because Gödel numbering worked at the level of the string, literally nothing that could be expressed in the system escaped his numbering, including that last string, which is meaningless.<br><br>Next, he showed how to use the system to build special statements called <em>meta-statements</em>, which referred to other statements. For example:<br>The statement “4 + 9 = 13” is a theorem.<br>The statement “4 + 9 = 13” is not a theorem.<br>The statement “4 + 9 = 13” includes the symbol “3.”<br>There exists a proof that the statement “4 + 9 = 13” is a theorem.<br><br>These meta-statements themselves were just statements, each with its own Gödel number. In this way, one meta-statement could refer to another meta-statement. A meta-statement could even refer to itself. For example:<br><br>“There does not exist a proof that this statement is a theorem.”<br><br>The paradox in the previous statement is even more subtle than it appears because Gödel built the statement to guarantee that at the level of semantics, the statement is a tautology. But consider the following:</span></p><ul><li><span style="font-family:times new roman,times,serif;">If the statement can be derived as a theorem, then it isn’t a theorem, so the contradiction of the statement can also be derived as a theorem, making the system by definition <em>inconsistent</em>.<br></span></li><li><span style="font-family:times new roman,times,serif;">If the statement can’t be derived as a theorem, then the system is by definition <em>incomplete</em>.</span></li></ul><p><span style="font-family:times new roman,times,serif;"><br>If it sounds complicated, that’s because it is. And we’ve only scratched the surface. Understanding <em>what</em> Gödel accomplished is fascinating but understanding <em>how</em> he accomplished it is something that even most mathematicians have only attained in a more or less sketchy fashion.<br><br><br>[1] As outlined by: Smullyan, Raymond M. 2014. “A Beginner’s Guide to Mathematical Logic”.
